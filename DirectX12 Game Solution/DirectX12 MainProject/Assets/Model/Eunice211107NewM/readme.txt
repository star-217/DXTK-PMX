モデルについての説明

【ボーン構造について】
　　このモデルでは、モデルの変形のために少し複雑なリグ構造を採用しています。
　　そのためボーンが多くて互換性が低いです。(古いMMDやMMD互換環境では動かない可能性あり）
　　リグ用のボーンはほとんど不可視にしてありますので、最新バージョンのMMDでお使いください。
　　動作についてはVer9.26（x64版）で確認しています。

　　準標準ボーンとして、
　　「全親」、「グルーブ」、「上半身2」（2は半角）、「肩P」、両腕の捩りボーン系、「腰」が入ってます。
　　
　　視線固定のためのボーン構造が入っています。
　　「両目IK」、「両目親」で注視点を操作できます。
　　この機能は「両目IK」をMMD上でOFFにすることで止めることができます。

　　いわゆる多段ボーン構造「MP2ボーン」の詳細はこちらで公開しているデータに含まれています。
　　（Lat式 http://www.nicovideo.jp/watch/sm17745051）
　　気になる方はチェックしてください。


【胸の物理設定について】
　　胸ボーンを三段化し、それぞれにRGBA(RGB-CMYK)さんのバランサー剛体を導入しました。
　　体の向きによって変形が苦しくなるのは今のところ仕様です。あと揺れ過ぎるかもしれません。


【モーフについて】
　　多くの目や口のモーフを多段化しています。
　　例えば、MMD標準的なモーフ「あ」「い」「う」「え」「お」はグループモーフとして設定されており、それぞれ
　　「ア」「イ」「ウ」「エ」「オ」と口内パーツを動かすボーンモーフが組み合わされて登録されています。
　　唇の開きに対して歯の閉じ加減をボーンモーフの作動量で自分好みに調整して設定することができます。
　　（PMXE等での作業が必要です）
　　自分好みのオフセット表情を作成し、口の開き具合や目の閉じ具合をデフォルトの状態から変化させたとき、
　　「まばたき」や「笑い」で目が閉じすぎたりすることがあります。
　　この場合、それぞれのグループモーフに含まれるモーフの作動量を調節することで、モーションに手を加えること
　　なく閉じすぎを防ぐことができます。



【目がキョロキョロ動く】
　　後ろ向きの体制の時に両目IKがONの状態となっていると、視線がIK制御を受けて両目が左右に頻繁に動く状態に
　　なります。回避するには視線のモーションを自作するか、両目IKをOFFにしてください。
　　それでなくとも目の動きがおかしいと思ったら、とりあえず両目IKをOFFにしてみるのが良いかもしれません。
